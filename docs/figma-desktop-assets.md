# Получение ассетов из Figma Desktop MCP

## Короткий вывод

Для текущего рабочего процесса с выделением в Figma Desktop не полагаться на `download_assets`: этот инструмент есть в официальном Figma MCP Cursor, но требует `fileKey + nodeId`. Локальный Figma Desktop MCP на `127.0.0.1:3845` удобен для текущего выделения, но `download_assets` не публикует.

Правильный путь для ассетов из текущей Figma:

1. Получить конкретные `nodeId` нужных фреймов через `get_metadata` локального Figma Desktop MCP.
2. Вызывать `get_design_context` отдельно для каждого фрейма, не для мультивыделения.
3. Из ответа достать временные URL вида `http://localhost:3845/assets/<hash>.<ext>`.
4. Скачать эти URL сразу, пока Figma Desktop MCP запущен.

## Почему не сработало сразу

Вызов `get_design_context` на мультивыделении возвращает sparse metadata. В таком ответе недостаточно полезного asset context, поэтому кажется, что прямых ассетов нет.

Если вызвать `get_design_context` по конкретному node id, например `8:341`, `8:164`, `8:232`, локальный MCP отдаёт ссылки на реальные PNG/SVG/JPG/WebP:

```text
http://localhost:3845/assets/<hash>.png
http://localhost:3845/assets/<hash>.svg
```

Такие ссылки можно скачивать напрямую через `urllib`, PowerShell или любой HTTP-клиент. Проверка PNG: HTTP `200`, первые байты `89504e470d0a1a0a`.

## Когда использовать скриншот

`get_screenshot` подходит как запасной вариант, когда нужен цельный рендер фрейма. Но это не замена скачиванию исходных ассетов: он сохраняет весь экран как один PNG и теряет структуру отдельных изображений.
